# Contributing to Atlas Bootstrap

Atlas Bootstrap follows the engineering standards defined in the atlas-handbook repository.

## Principles

- Keep changes small and focused.
- Prefer simple, readable shell scripts.
- Make scripts safe to run more than once where practical.
- Document meaningful behavior in the repository.
- Use Git history to explain both what changed and why.

## Workflow

Before committing, review your changes:

```bash
git status
git diff
```

Commit messages should clearly describe the purpose of the change.

## Scope

Atlas Bootstrap should remain focused on provisioning Atlas Linux systems.

Project philosophy, architecture, standards, and historical records belong in the atlas-handbook repository.
